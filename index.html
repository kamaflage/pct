Tips
File namingName bilingual HTML files with consistent suffixes matching your Settings hints — e.g. article_en.html + article_tgt.html. The app uses these to auto-detect which file is source and which is target.
AlignmentJW.org publications auto-align by paragraph number (data-pid). Other HTML falls back to positional alignment — always review in the studio before pushing.
BackupAll data is stored in your browser's localStorage. Clearing browser data wipes everything. Export regularly and keep the file somewhere safe.
Import safetyImporting the same file twice is safe — duplicates are detected and skipped. The app reports how many pairs and phrases were added vs skipped.
New here? Follow these steps to go from zero to your first translation.
1
Set up your languages
Tap ⚙ in the topbar. Enter your target language name (e.g. Plautdietsch), an ISO code if you know it, and the file suffix your bilingual files use (e.g. _pdt). Save.
2
Add content to your corpus
Go to Builder. Choose a method:
• Files — pick one English + one target HTML file
• Folder — pick a folder of paired HTML files
• Paste Text — paste an article directly in both languages
The app opens Alignment Studio — review the paragraph pairs, remove mismatches, then Accept & Push.
3
Build your phrase library
Go to Phrases. Tap 🔍 Suggestions to see auto-extracted word pairs from your corpus. Review them — tap Add ✅ to keep or Skip ✕ to discard. Use Add ≥80% to approve all high-confidence suggestions at once.
4
Import a dictionary (optional)
If you have a bilingual dictionary, ask an AI to convert it to the PCT import format (see the Dictionary Import tab). Import the resulting .txt file via Corpus → Load .txt. It will appear as a filter chip in the Phrases tab.
5
Translate something
Go to Translate. Paste your English source text, tap Analyze then Build Prompt. Copy the prompt into Claude, ChatGPT, or any LLM. Paste the response back, tap Enter Review Mode, approve each segment, and Save.
6
Back up your work
Tap ⬇ Export in the topbar. Save the downloaded .txt file. This file contains everything — corpus pairs, phrase library, all source tags. You can import it on any device to restore your full workspace.
Any bilingual dictionary or word list can be imported as a tagged phrase source. Once imported, it appears as its own filter chip in the Phrases tab.
Import File Format
Header File must begin with:
PDT CORPUS EXPORT
Exported: YYYY-MM-DD
Pairs: 0
Phrases: [count]

## CORPUS PAIRS

## PHRASE LIBRARY
Each entry TERM: English word or phrase
PLAUTDIETSCH: Target translation
CATEGORY: common | CONFIDENCE: verified | SOURCE: source-name
---

Note: the field is literally PLAUTDIETSCH: regardless of language — this is a fixed parser keyword.
SOURCE key Use lowercase with hyphens — e.g. herman-rempel, jw-glossary, smith-dict. All entries from the same dictionary must share the same SOURCE value. This becomes the filter chip label (auto-capitalized).
CATEGORY Valid values: common, religious, grammar, proper. Use religious for Bible terms, congregation vocabulary, and proper names.
CONFIDENCE verified — confirmed correct. suggested — needs review. Dictionary imports should use verified.
Asking an AI to Build a Dictionary File
Prompt Give the AI the dictionary content (paste it, upload it, or share the URL) and say:

"Convert this dictionary to PCT Translator import format. Use SOURCE: [your-source-name]. Output a plain text file starting with the required header. Each entry must use TERM:, PLAUTDIETSCH:, CATEGORY:, CONFIDENCE:, SOURCE:, and --- separator."

Include this help tab's Entry format as a reference so the AI knows the exact structure.
After import Go to Phrases — your new source chip will appear automatically in the filter bar. Tap it to view only entries from that dictionary.
